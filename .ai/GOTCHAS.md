# xhttp.lol — Gotchas

> Things that bite you if you forget them.

## 1. SPA is in handler, not a separate build

The HTML is generated by `spa.ts` and served by the Worker directly — there's no separate client build step, no Vite, no build pipeline. To change the HTML shell, edit `spa.ts`.

## 2. Real Chrome UA for probing

`constants.ts` uses a real Chrome User-Agent string for all external fetches. This is intentional — WAFs and CDNs serve different responses to bot UAs. If you change it to a bot-style UA, scan results will differ from what real browsers see.

## 3. Package name mismatch

`package.json` still says `"name": "preflight-lol"` from before the rename. Functionally irrelevant (private package) but confusing.

## 4. Shared probe dependency

TLS version detection uses the Yoke Fly probe (`PROBE_URL`). If the probe is down or the auth key is wrong, TLS data comes back null. This is by design — TLS is informational, not critical to the scan grade.

## 5. install.sh redirect still points to old repo name

`/install.sh` redirects to `https://raw.githubusercontent.com/yokedotlol/preflight/main/cli/install.sh`. If the repo was renamed on GitHub, this URL may 404.

## 6. CSP nonce on HTML responses

Every HTML response gets a fresh `crypto.randomUUID()` nonce for CSP. The SPA's inline scripts must use this nonce. If you add any `<script>` tag to `spa.ts`, include the nonce or it'll be blocked by CSP.

## 7. Rate limiter fails open

If the Durable Object rate limiter throws, the request is allowed through (empty catch block in `checkRateLimit`). This is intentional — availability over strict enforcement.

## 8. Cache key includes sub-route

Cache key format is `scan:{domain}:{route}` where route is 'full', 'cors', 'csp', 'headers', 'chain', or 'cache'. A full scan doesn't populate sub-route caches and vice versa.
